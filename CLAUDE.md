# CLAUDE.md — PPC Agent Apollo Next

Перед будь-якою роботою прочитай INSTRUCTIONS.md повністю.

---

## Що це за проект
Мультиагентна система автоматизації запуску рекламних кампаній Meta Ads
для мережі фітнес-клубів Apollo Next. Управління через Telegram.

---

## Критичні правила

1. **PAUSED завжди** — всі об'єкти в Meta API створюються зі статусом PAUSED
2. **Публікація тільки після апруву** — явне "ок" або "запускай" від користувача
3. **.env не коммітити** — токени тільки локально, ніколи в GitHub
4. **Читай rules.md** — перед будь-якою роботою з неймінгом або посиланнями
5. **Один агент = одна задача** — не суміщай ролі агентів

---

## Структура агентів

| Файл | Роль | Статус |
|------|------|--------|
| agents/naming_agent.md | Генерує назви кампаній/груп/об'явлень | ✅ готов |
| agents/utm_agent.md | Генерує UTM розмітку | ✅ готов |
| agents/link_checker.md | Перевіряє посилання до публікації | ✅ готов |
| agents/validator.md | Збирає структуру і показує превью | ✅ готов |
| agents/publisher.md | Публікує через Meta API | ✅ готов |

---

## Флоу запуску кампанії

```
ТЗ від користувача
  → naming_agent (JSON з назвами)
  → utm_agent (JSON + UTM)
  → link_checker (перевірка посилань)
       ❌ помилка → стоп
       ✅ OK → далі
  → validator (превью для апруву)
       ← апрув від користувача
  → publisher (створення в API, статус PAUSED)
  → запис в history/launches.json
  → підтвердження користувачу
```
