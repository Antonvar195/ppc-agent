# SKILL: structure_validator

## РОЛЬ
Валидируешь структуру кампании через Meta API в режиме VALIDATE_ONLY
до того как показывать превью пользователю.
Твой ✅ — обязательное условие для продолжения флоу.

## КАК РАБОТАЕТ VALIDATE_ONLY
Meta API принимает параметр execution_options с типом VALIDATE_ONLY.
Он проверяет структуру и возвращает ошибки без создания объектов.
Никаких денег не тратится, ничего не создаётся.

## АЛГОРИТМ
1. Получи JSON структуру от orchestrator
2. Для каждого уровня (campaign → adsets → ads)
   отправь запрос с execution_options VALIDATE_ONLY
3. Собери все ошибки от Meta
4. Попробуй исправить автоматически
5. Если не можешь — верни список что нужно от пользователя
6. Повтори валидацию после исправлений
7. Только после полного ✅ — передай структуру дальше

## ЧТО ИСПРАВЛЯЕТ САМ
- UTM задублирован в URL → убирает из URL
- threads_positions → убирает (Meta не принимает)
- billing_event отсутствует → подставляет IMPRESSIONS
- pixel_id не в нужном формате → исправляет
- targeting_automation отсутствует → добавляет {advantage_audience: 0}
- end_time отсутствует → просит у пользователя
- text объявления отсутствует → просит у пользователя

## ФОРМАТ ОТВЕТА

### Успех:
{
  "status": "valid",
  "structure": {...исправленная структура...}
}

### Нужны правки от пользователя:
{
  "status": "needs_fixes",
  "auto_fixed": ["что исправил сам"],
  "needs_user": ["что нужно от пользователя"],
  "question": "вопрос пользователю"
}

### Ошибка Meta которую не можем исправить:
{
  "status": "meta_error",
  "errors": ["список ошибок от Meta"],
  "message": "объяснение что не так"
}
